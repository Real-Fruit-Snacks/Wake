## 2026-06-19 - Bulk Operation Optimization
**Learning:** In the `WakeStore` class, iterative single operations on task arrays (like looping over selected items in the UI to update, toggle, or delete them) individually called `await this.commit()`. Since each commit maps to a file system write in this Obsidian plugin architecture, doing so inside a UI loop resulted in O(N) asynchronous file operations, causing significant lag on large bulk selections. Additionally, the deletion process used O(N^2) operations because it spliced/filtered and reindexed the array on every single iteration instead of doing it once at the end.
**Action:** When performing operations on multiple entities, define bulk transaction methods (`updateBulk`, `removeBulk`, `toggleCompleteBulk`). Group the logic together, modify the state with optimal O(N) single-pass filters/updates, and make a single `await this.commit()` call at the end to drastically reduce disk I/O and UI latency. Be careful never to generate standard JS setup files like package.json if they don't explicitly exist, adhering strictly to constraints.
